## MoveIt2 테스트 상세 결과

### 환경 구성
- Launch: `ros2 launch dsr_bringup2 dsr_bringup2_moveit.launch.py mode:=virtual host:=127.0.0.1 port:=12345 model:=m1013`
- Gazebo 없이 MoveIt2 단독 모드 (FakeSystem 하드웨어)
- RViz 17fps (WSL2 소프트웨어 렌더링)

### FK (Forward Kinematics) 검증
- Home Position (모든 조인트 0도)의 엔드이펙터 위치:
  - Position: x=0.0001, y=0.0348, z=1.4525 [m]
  - Orientation: qw=1.0 (거의 단위 쿼터니언)

### 조인트 공간 모션 플래닝 (OMPL)
- 목표: J1=45°, J2=-30°, J3=60°, J4=0°, J5=-60°, J6=30°
- Plan Only: 0.04초 만에 45개 경로 포인트 생성 (예상 실행시간 4.36s)
- Plan & Execute: 4.55초에 실행 완료, 목표 대비 오차 < 1°
- Home 복귀: 4.89초에 완료, 오차 < 0.5°

### 사용 가능 플래너
| 플래너 | 용도 |
|--------|------|
| OMPL (기본) | 샘플링 기반, 범용 모션 플래닝 |
| CHOMP | 최적화 기반, 장애물 회피에 강점 |
| Pilz (PTP/LIN/CIRC) | 산업용 정밀 모션 (직선, 원호) |

## 주요 발견 사항
- `dsr_bringup2_moveit.launch.py`는 Gazebo 없이 동작 (ros2_control FakeSystem 사용)
- 실제 Gazebo 시뮬레이션과 연동하려면 `dsr_bringup2_gazebo.launch.py`와 별도 구성 필요
- MoveGroup Action (`/move_action`)으로 Python에서 직접 제어 가능
- 컨트롤러 순서: joint_state_broadcaster → dsr_controller2 → dsr_moveit_controller (자동 시퀀싱)
- RViz 인터랙티브 마커로 GUI에서도 목표 포즈 설정 가능
