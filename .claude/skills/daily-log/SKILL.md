# 프로젝트 일지 스킬

## 개요
하루 작업 단위로 진행 상황을 기록하는 일지를 작성/관리한다.
세션 간 연속성을 유지하고, 프로젝트 히스토리를 추적하는 데 활용한다.

## 저장 위치
```
workspace/daily-logs/
├── 2026-03-03.md
├── 2026-03-04.md
└── ...
```

## 일지 작성 규칙

### 트리거 시점
- 사용자가 "일지 작성", "오늘 정리", "마무리" 등을 요청할 때
- 하루 작업 종료 시 사용자가 마무리 의사를 표현할 때

### 작성 절차
1. 해당 날짜 파일이 이미 있으면 **업데이트** (덮어쓰기 아님, 내용 추가)
2. 없으면 **새로 생성**
3. 작업 중 변경/생성된 파일 목록을 자동으로 수집
4. 현재 대화에서 수행한 작업을 요약

### 작성 형식

```markdown
# 프로젝트 일지 — {YYYY-MM-DD}

## 오늘 완료한 작업
- [ 완료 항목 1 ]
- [ 완료 항목 2 ]

## 주요 결정 사항
- [ 기술적 결정이나 방향 전환 등 ]

## 발생한 이슈 & 해결
- [ 이슈 → 해결 방법 ] (상세 내용은 lessons-learned.md 참조)

## 변경된 파일
- [ 파일 경로 — 변경 내용 요약 ]

## 내일 이어서 할 작업
- [ 다음 작업 1 ]
- [ 다음 작업 2 ]

## 메모
- [ 기타 기록할 사항 ]
```

### 연계 규칙
- 기술적 트러블슈팅 → `knowledge-base/lessons-learned.md`에도 기록
- 설치/환경 변경 → `knowledge-base/installation-guide.md` 업데이트
- 프로젝트 구조 변경 → `PROJECT_STRUCTURE.md` 업데이트
- 일지는 **요약 기록**, 상세 내용은 위 문서들에 분산 저장
